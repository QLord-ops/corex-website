# Конструктор сайтов для клиентов COREX

## Идея

Клиенты, которые приходят на corex-website, получают доступ к **конструктору сайтов**: могут собрать свою лендинг-страницу из готовых блоков (как сцены на основном сайте) без кода, сохранить и получить публичную ссылку.

## Роли

- **Клиент** — компания/пользователь с доступом в конструктор (создание и редактирование своих сайтов).
- **Публика** — просмотр опубликованного сайта клиента по ссылке.

## Модель данных

### Клиент (Client)
- `id` (uuid)
- `name` — название компании/клиента
- `email` — для входа и уведомлений (опционально на первом этапе)
- `created_at`

### Сайт клиента (ClientSite)
- `id` (uuid)
- `client_id` — владелец
- `name` — внутреннее название (например «Лендинг продукта X»)
- `slug` — уникальный URL (например `acme-demo` → просмотр на `/s/acme-demo`)
- `blocks` — массив блоков (см. ниже)
- `published_at` — дата публикации; `null` = черновик
- `created_at`, `updated_at`

### Блок (Block)
Каждый элемент страницы — блок с типом и настраиваемыми полями:

| Тип блока   | Описание           | Пример полей (props) |
|------------|--------------------|-----------------------|
| `hero`     | Главный экран      | title, subtitle, ctaText, ctaLink |
| `pain`     | Боли/проблемы      | title, points[]       |
| `how`      | Как мы работаем    | title, subtitle, steps[] |
| `proof`    | Результаты/цифры   | title, subtitle, stats[] |
| `decision` | Переход к действию | title, subtitle       |
| `cta`      | Призыв к действию  | title, buttonText, buttonLink |
| `faq`      | Вопросы-ответы     | title, items[]        |
| `testimonials` | Отзывы         | title, items[]        |
| `about`    | О компании         | title, body           |

Данные блока хранятся как: `{ type: "hero", props: { title: "...", ... } }`.

## Маршруты

### Основной сайт COREX
- `/` — текущий лендинг (ScrollExperience).

### Конструктор (для клиентов)
- `/builder` — дашборд: список сайтов клиента, кнопка «Создать сайт».
- `/builder/sites/:siteId` — редактор сайта: добавление/удаление/перестановка блоков, редактирование полей каждого блока.
- (Позже) `/builder/settings` — профиль клиента.

### Публичный просмотр сайта клиента
- `/s/:slug` — рендер опубликованного сайта по `slug` (только если `published_at` задан).

## Технические решения

1. **Бэкенд (FastAPI + MongoDB)**  
   Коллекции: `clients`, `client_sites`. API: CRUD по клиентам и сайтам, получение опубликованного сайта по `slug`.

2. **Фронтенд (React)**  
   - Роутинг: `react-router-dom` (уже есть).  
   - Конструктор: список сайтов → редактор с панелью блоков слева, превью/канвас по центру, панель свойств справа (или снизу на мобильных).  
   - Просмотр `/s/:slug`: один компонент `ClientSiteView`, который по данным `blocks` рендерит известные типы блоков (те же визуальные паттерны, что и сцены на главной).

3. **Авторизация (этап 1)**  
   Минимальный вариант: доступ в `/builder` по `client_id` в query или простой форме «введите ваш ключ/email». Позже — JWT или сессии.

4. **Публикация**  
   Кнопка «Опубликовать» в редакторе выставляет `published_at`; ссылка вида `https://corex.../s/{slug}` становится доступной.

## Этапы внедрения

- **Этап 1 (текущий):** API клиентов и сайтов, дашборд конструктора (список сайтов + создание), просмотр по `/s/:slug` с захардкоженным или выбранным клиентом.
- **Этап 2:** Редактор блоков (добавление, порядок, редактирование props).
- **Этап 3:** Авторизация клиентов, привязка сайтов к клиенту.
- **Этап 4:** Темы, домены, аналитика — по необходимости.
